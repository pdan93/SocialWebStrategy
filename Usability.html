Usability Test Plan Section

Test Plan for: Twitter Fight Component

Persona Type Tested: Primary/Secondary/Negative/Served

Context: the user tested the game at home, at his own pace
Duration: 15 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives

Quality and Overall Playability of the Game etc.).

  • Make sure that the interface is light, yet intuitive, responding to user feedback;
  • Keep the game simple, yet fun and pleasent;
  • Have a basic idea of what features can be added or improved upon;
Scenario

Imagine you are looking over what some of your friends did or talked about recently and as you finish you would like to unwind and see what other people they follow. This game is a solution as it arranges famous people your friend follows in a quick list and you can see them as you fight out the match..

Tasks

Each user will have to go through these task in the estimated time. If they take longer, as this is a game of 1 vs 1 it will not be enjoyable for the other party.

  • Task1-1: Logging in/Interface accomodation + estimated 30-60 seconds.;
  • Task1-2: Choosing a character + 60 seconds;
  • Task2-1:Setting up the board + limited by timer 60 seconds;
  • Task2-2:Actual Gameplay + between 5 to 15 minutes;

If a user completes a task faster, then it is better to have ideas of designing a ranking system as part of the interface.

Post-test Questions

These questions could be presented to the user as a questionnaire or verbally asked.

  • Did you find the game fun? Would you replay it?
  • What did you think about the interface? Was it to your liking? Did it lack anything?
  • Was the design easy to comprehend/intuitive? Was it difficult to play?
  • As many questions needed.
Notes for Test Coordinator
  • The tasks are all times based. The users basically are monitored as they perform the tasks.;
  • Read the user the scenario for the test;
  • The tasks are straight forward. If the user shows sign of confusion, then the interface should be simplified;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).

Test Results for: Twitter Fight

Success Criteria

A successful design has been achieved when:

  • 50% of users have managed to play a game to the end within the time frames allocated above;
  • 80% of users have managed to play a game to the end;
  • The game complexity has been satisfactory to 50% of the end users;

Summary

After the tests the user gave the following feedback

  • 3 out of 4 users found that the game is too simple and wanted more complex features;
  • 2 out of 4 users have found the game short, but fun;
  • 3 out of 4 users used in test have enjoyed the interface and likes how simple it is to navigate;
  • These finding reflect the observation notes of the observer.
